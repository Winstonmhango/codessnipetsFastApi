Getting Started

The CodeSnippets API provides endpoints for managing blog posts, categories, series, booklets, learning paths, and user authentication.

Authentication

The API uses JWT (JSON Web Tokens) for authentication. Register a new user or login to get an access token.

POST /api/v1/auth/register
POST /api/v1/auth/login

Blog Posts

Create, read, update, and delete blog posts. Get posts by category, author, or find related posts.

GET /api/v1/posts
GET /api/v1/posts/{slug}

Series

Manage series and their articles. Create new series, add articles, and find related series.

GET /api/v1/series
GET /api/v1/series/{slug}

Booklets

Manage booklets, chapters, and updates. Create new booklets, add chapters, and track updates.

GET /api/v1/booklets
GET /api/v1/booklets/{slug}

Learning Paths

Create and manage learning paths with modules, content items, and resources.

GET /api/v1/learning-paths
GET /api/v1/learning-paths/{slug}

Categories

Manage categories for blog posts. Create, update, and delete categories.

GET /api/v1/categories
GET /api/v1/categories/{slug}

Integration with Next.js Frontend

The API is designed to work seamlessly with the CodeSnippets Next.js frontend. Check the documentation for detailed integration examples.

Example API Client

// lib/api-client.ts
const API_URL = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8000/api/v1';

export async function fetchAPI(endpoint, options = {}) {
  const token = localStorage.getItem('token');
  
  const headers = {
    'Content-Type': 'application/json',
    ...options.headers,
  };
  
  if (token) {
    headers['Authorization'] = `Bearer ${token}`;
  }
  
  const response = await fetch(`${API_URL}${endpoint}`, {
    ...options,
    headers,
  });
  
  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.detail || 'An error occurred');
  }
  
  return response.json();
}
